* Environment
User-level environment variables for =systemd --user= and login shells, set via
[[https://www.freedesktop.org/software/systemd/man/environment.d.html][=environment.d=]]. The user manager imports these at startup, so every user
service spawned afterwards inherits them — no need to repeat them in each unit
file. Changes require a re-login (or =systemctl --user daemon-reexec= +
restart of the affected services).

This is the central registry for =HARIS_*= vars referenced from services and
scripts. Keep entries here rather than hard-coding values in individual units.
#+begin_src conf :tangle ~/.config/environment.d/10-haris.conf
  # Host port the Excalidraw canvas docker service publishes on 127.0.0.1.
  # Referenced by excalidraw-canvas.service and by ai.org's MCP registration.
  HARIS_EXCALIDRAW_PORT=10002

  # Host port for the webscripts service.
  HARIS_WEBSCRIPTS_PORT=10001
#+end_src

** Tmux Server
Run the tmux server as its own user service so it lives in its own cgroup
instead of inheriting the cgroup of whichever terminal/scope first launched it.
That isolates it from systemd-oomd policies on =app.slice= and from
=MemoryHigh= ceilings on launcher scopes (e.g. the VSCode wrapper).
#+begin_src systemd :tangle ~/.config/systemd/user/tmux.service
  [Unit]
  Description=Tmux server
  PartOf=default.target
  Wants=tmux-shell-cleanup.timer

  [Service]
  SyslogIdentifier=tmux
  Type=forking
  OOMScoreAdjust=-700
  ManagedOOMPreference=omit
  Environment=XDG_RUNTIME_DIR=/run/user/%U
  ExecStart=/usr/bin/env tmux new-session -d -s default
  ExecStop=/usr/bin/env tmux kill-server
  Restart=on-failure

  [Install]
  WantedBy=default.target
#+end_src
** Tmux Shell Cleanup
Periodically kill detached =term-*= sessions created by [[file:scripts.org::*=tmux-shell=][=tmux-shell=]].
#+begin_src systemd :tangle ~/.config/systemd/user/tmux-shell-cleanup.service
  [Unit]
  Description=Clean detached tmux-shell (term-*) sessions
  After=tmux.service

  [Service]
  SyslogIdentifier=tmux-shell-cleanup
  Type=oneshot
  Environment=XDG_RUNTIME_DIR=/run/user/%U
  ExecStart=/bin/sh -c 'tmux list-sessions -F "#{session_attached}:#{session_name}" 2>/dev/null | sed -n "s/^0:\\(term-.*\\)/\\1/p" | xargs -r -n1 tmux kill-session -t'
#+end_src
#+begin_src systemd :tangle ~/.config/systemd/user/tmux-shell-cleanup.timer
  [Unit]
  Description=Clean detached tmux-shell (term-*) sessions every 15 minutes

  [Timer]
  OnBootSec=15min
  OnUnitActiveSec=15min
  Unit=tmux-shell-cleanup.service

  [Install]
  WantedBy=timers.target
#+end_src
** Excalidraw canvas
Docker container hosting the live Excalidraw canvas. The stdio MCP server (see
[[file:ai.org::*MCP servers][ai.org]]) syncs into it via =EXPRESS_SERVER_URL=. View the canvas at
=http://127.0.0.1:$HARIS_EXCALIDRAW_PORT= (see [[Environment]] for the port).

The container binds inside to =0.0.0.0:3000= (=HOST=0.0.0.0= overrides its
loopback default so port-forwarding works) but is published to the host only
on =127.0.0.1:$HARIS_EXCALIDRAW_PORT= so it's not reachable off the machine.
#+begin_src systemd :tangle ~/.config/systemd/user/excalidraw-canvas.service
  [Unit]
  Description=Excalidraw canvas (MCP)
  PartOf=default.target

  [Service]
  SyslogIdentifier=excalidraw-canvas
  ExecStartPre=-/usr/bin/docker pull ghcr.io/yctimlin/mcp_excalidraw-canvas:latest
  ExecStartPre=-/usr/bin/docker rm -f mcp-excalidraw-canvas
  ExecStart=/usr/bin/docker run --rm --name mcp-excalidraw-canvas -e HOST=0.0.0.0 -p 127.0.0.1:${HARIS_EXCALIDRAW_PORT}:3000 ghcr.io/yctimlin/mcp_excalidraw-canvas:latest
  ExecStop=/usr/bin/docker stop mcp-excalidraw-canvas
  Restart=on-failure
  TimeoutStartSec=300

  [Install]
  WantedBy=default.target
#+end_src

#+begin_src shell :tangle ~/.local/bin/crontab-listen-update
  if ! which inotifywait 2>/dev/null; then
      echo "ERROR: inotifywait is not installed. Aborting."
      exit 1
  fi

  mkdir -p ~/.cron.d/
  while :; do
      content=''
      for file in ~/.cron.d/*; do
          echo "Read crontab file: $file"
          content="$(
              echo "$content"
              echo
              echo "############## START $file"
              echo
              cat "$file"
              echo
              echo "############## END  $file"
          )"
      done
      echo "$content" | crontab -
      echo "Resulting content of crontab:"
      crontab -l
      echo
      inotifywait -e modify -e attrib -e create -e delete ~/.cron.d/
  done
#+end_src

** Clipboard Transformer
Watches the clipboard and transforms its contents based on which pattern the
existing content matches. New transformations can be added as additional
branches in the dispatch =case=.

  [Service]
  SyslogIdentifier=clipboard-transformer
  ExecStart=/usr/bin/env clipboard-transformer
#+end_src
*** Executable
#+begin_src bash :tangle ~/.local/bin/clipboard-transformer
  transform() {
      local cb="$1"
      # Jira branch: "git checkout -b TICKET-123" -> just the branch name.
      if [[ "$cb" =~ ^git[[:space:]]+checkout[[:space:]]+-b[[:space:]]+[A-Z]+-[0-9]+ ]]; then
          rev <<<"$cb" | cut -f1 -d' ' | rev
          return
      fi
      # Strip a leading shell prompt ("$ " or " $ ") from a single-line clipboard.
      if [[ "$(wc -l <<<"$cb")" -le 1 ]] && [[ "$cb" =~ ^[[:space:]]*\$[[:space:]]+ ]]; then
          sed -E 's/^[[:space:]]*\$[[:space:]]+//' <<<"$cb"
          return
      fi
      # Strip a leading "● " or "❯ " prompt marker from the first line only.
      # Trailing whitespace may be a non-ASCII space (NBSP, narrow no-break,
      # etc.), so match Unicode whitespace via Perl.
      if [[ "$cb" == ●* || "$cb" == ❯* ]]; then
          perl -CSDA -pe 's/^[●❯]\p{Space}+// if $. == 1' <<<"$cb"
          return
      fi
      return 1
  }

  while :; do
      echo "Scanning clipboard..."
      cb="$(timeout 1s xsel -bo)"
      if new="$(transform "$cb")" && [ -n "$new" ] && [ "$new" != "$cb" ]; then
          echo "Clipboard matched."
          echo "Transformed content:"
          sed 's/^/  /' <<<"$new"
          echo -n "$new" | xsel -b
      fi
      sleep 3
  done
#+end_src
* Out-of-memory handling
When VSCode (or anything else) eats enough RAM to push the system into swap
thrashing, the X server stops responding and the only escape is magic-sysrq
OOM, which often kills innocent processes.

The mitigation has three layers:
1. ~OOMScoreAdjust~ on critical services (i3, dunst, sxhkd) so the kernel OOM
   killer never picks them as victims, plus ~ManagedOOMPreference=omit~ on the
   recovery-critical subset (i3, sxhkd, sxhkd-private, dunst) so systemd-oomd
   also skips them. Note: ~OOMScoreAdjust~ alone does not protect from
   systemd-oomd, which selects by cgroup-level PSI/swap pressure;
   ~ManagedOOMPreference=omit~ is what excludes a unit from oomd's candidate set.
2. ~systemd-oomd~ at the system level watches PSI and swap usage and kills the
   heaviest cgroup before the system locks up. System-side config lives in
   [[file:sys.org::*systemd-oomd][sys.org]].
3. A VSCode wrapper that runs it inside its own cgroup with a memory ceiling and
   a high OOM score, so it gets throttled/killed first.

** Opt user services into systemd-oomd
Tells oomd it's allowed to act on processes inside =app.slice= (where
=systemd-run --user= places transient scopes, and where graphical apps
launched via desktop files end up).
#+begin_src conf :tangle ~/.config/systemd/user/app.slice.d/50-oomd.conf
  [Slice]
  ManagedOOMSwap=kill
  ManagedOOMMemoryPressure=kill
  ManagedOOMMemoryPressureLimit=50%
#+end_src
** VSCode memory-bounded launcher
The =code= wrapper lives in [[file:scripts.org::*=code=][scripts.org]]. It runs VSCode inside a
transient systemd scope under =app.slice= with a ~MemoryHigh~ ceiling and a
high ~OOMScoreAdjust~ so it's the first thing killed under memory pressure.
